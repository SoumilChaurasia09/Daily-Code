# 0002. Add Two Numbers

**Problem Name:** Add Two Numbers  
**Difficulty:** Medium

---

## Question

You are given two non-empty linked lists representing two non-negative integers.

The digits are stored in reverse order, where each node contains a single digit.

Add the two numbers and return the result as a linked list in the same reversed format.

---

## My Solution

```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */

class Solution {
public:
    ListNode* addTwoNumbers(ListNode* l1, ListNode* l2) {

        ListNode* dummy = new ListNode(0);
        ListNode* temp = dummy;

        int carry = 0;

        while (l1 != nullptr || l2 != nullptr || carry != 0) {

            int sum = carry;

            if (l1 != nullptr) {
                sum += l1->val;
                l1 = l1->next;
            }

            if (l2 != nullptr) {
                sum += l2->val;
                l2 = l2->next;
            }

            carry = sum / 10;

            temp->next = new ListNode(sum % 10);
            temp = temp->next;
        }

        return dummy->next;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(max(n, m))` |
| **Space Complexity** | `O(max(n, m))` *(For the output linked list.)* |

---

## What I Learned

This problem introduced me to working with **Linked Lists**.

I learned how to traverse two linked lists simultaneously while handling a carry, just like adding two numbers on paper. A dummy node made it easier to build the answer list without worrying about the first node separately.

---

## Personal Notes

Initially, I thought about converting the linked lists into integers, adding them, and then converting the result back into a linked list. However, that approach doesn't work for very large numbers because the lists can contain up to 100 digits.

Building the answer node by node while maintaining a carry turned out to be a much better approach. This problem also helped me become more comfortable with linked list traversal and node creation.
