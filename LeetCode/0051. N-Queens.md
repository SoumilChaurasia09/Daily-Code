# 0051. N-Queens

**Problem Name:** N-Queens  
**Difficulty:** Hard

---

## Question

The **N-Queens** puzzle requires placing `n` queens on an `n × n` chessboard such that no two queens attack each other.

A queen can attack another queen if they share the same:

- Row
- Column
- Diagonal

Return all distinct valid board configurations.

---

## My Solution

```cpp
class Solution {
public:
    vector<vector<string>> ans;

    bool isSafe(vector<string>& board, int row, int col, int n) {

        for (int i = row - 1; i >= 0; i--) {
            if (board[i][col] == 'Q')
                return false;
        }

        for (int i = row - 1, j = col - 1; i >= 0 && j >= 0; i--, j--) {
            if (board[i][j] == 'Q')
                return false;
        }

        for (int i = row - 1, j = col + 1; i >= 0 && j < n; i--, j++) {
            if (board[i][j] == 'Q')
                return false;
        }

        return true;
    }

    void solve(vector<string>& board, int row, int n) {

        if (row == n) {
            ans.push_back(board);
            return;
        }

        for (int col = 0; col < n; col++) {

            if (isSafe(board, row, col, n)) {

                board[row][col] = 'Q';

                solve(board, row + 1, n);

                board[row][col] = '.';
            }
        }
    }

    vector<vector<string>> solveNQueens(int n) {

        vector<string> board(n, string(n, '.'));

        solve(board, 0, n);

        return ans;
    }
};
```

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(N!)` |
| **Space Complexity** | `O(N²)` |

## What I Learned

This problem introduced me to the concept of **Backtracking**, where decisions are made one step at a time and undone whenever they lead to an invalid state.

I learned how to recursively place queens row by row while checking whether each position is safe. If a valid position is found, I move to the next row. Otherwise, I backtrack by removing the queen and trying another position.

This problem also helped me understand recursion, state restoration, and constraint checking.

## Personal Notes

At first, this problem looked intimidating because it was my first Hard-level question.

The biggest challenge wasn't writing the code—it was understanding the recursive thought process. Once I realized that every row can be solved independently by trying every column and backtracking whenever necessary, the solution became much clearer.

This problem gave me a much better understanding of how recursive search algorithms work and why backtracking is useful for solving constraint-based problems.
