# 0350. Intersection of Two Arrays II

**Problem Name:** Intersection of Two Arrays II  
**Difficulty:** Easy

---

## Question

Given two integer arrays `nums1` and `nums2`, return an array representing their intersection.

Each element in the result should appear as many times as it appears in **both** arrays.

The order of the result does not matter.

---

## My Solution

```cpp
class Solution {
public:
    vector<int> intersect(vector<int>& nums1, vector<int>& nums2) {

        unordered_map<int, int> mp;

        for (int i = 0; i < nums1.size(); i++) {
            mp[nums1[i]]++;
        }

        vector<int> ans;

        for (int i = 0; i < nums2.size(); i++) {

            if (mp[nums2[i]] > 0) {
                ans.push_back(nums2[i]);
                mp[nums2[i]]--;
            }
        }

        return ans;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n + m)` |
| **Space Complexity** | `O(n)` |

Where:
- `n` = Size of `nums1`
- `m` = Size of `nums2`

---

## What I Learned

This problem helped me understand how to use a **Hash Map** to count the frequency of elements.

I first stored the frequency of each number from the first array. Then, while traversing the second array, I checked whether the current number was still available in the map. If it was, I added it to the answer and decreased its frequency.

This approach correctly handles duplicate values.

---

## Personal Notes

Initially, I thought about using a set like in the previous intersection problem. However, a set removes duplicate values, while this problem requires duplicates to appear in the answer as many times as they exist in both arrays.

Using a hash map to store frequencies made the solution simple and efficient. This problem also reinforced the difference between finding **unique common elements** and finding **all common occurrences**.
