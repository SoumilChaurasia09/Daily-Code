# 0088. Merge Sorted Array

**Problem Name:** Merge Sorted Array  
**Difficulty:** Easy

---

## Question

You are given two sorted integer arrays `nums1` and `nums2` along with their valid sizes `m` and `n`.

Merge both arrays into a single sorted array. The merged result should be stored inside `nums1`, which has enough extra space to hold all elements.

---

## My Solution

```cpp
class Solution {
public:
    void merge(vector<int>& nums1, int m, vector<int>& nums2, int n) {

        int i = m - 1;
        int j = n - 1;
        int k = m + n - 1;

        while (i >= 0 && j >= 0) {

            if (nums1[i] > nums2[j]) {
                nums1[k] = nums1[i];
                i--;
            }
            else {
                nums1[k] = nums2[j];
                j--;
            }

            k--;
        }

        while (j >= 0) {
            nums1[k] = nums2[j];
            j--;
            k--;
        }

    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(m + n)` |
| **Space Complexity** | `O(1)` |

---

## What I Learned

This problem introduced me to the idea of merging arrays from **right to left**.

Instead of shifting elements or creating another array, I can compare the largest elements of both arrays and place the greater one at the end of `nums1`. Since the extra space already exists at the end, no existing values are overwritten.

This approach keeps the solution efficient and avoids using additional memory.

---

## Personal Notes

Initially, I thought about copying all elements into a new array and sorting them. Although that works, it uses extra space and doesn't satisfy the follow-up requirement.

The key insight was to start filling the array from the last index because that's where the free space already exists.

This problem helped me understand how changing the direction of traversal can simplify an in-place array problem.
