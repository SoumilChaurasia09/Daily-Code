# 0283. Move Zeroes

**Problem Name:** Move Zeroes  
**Difficulty:** Easy

---

## Question

Given an integer array `nums`, move all `0`s to the end while maintaining the relative order of the non-zero elements.

The operation must be performed **in-place**, without using an extra array.

---

## My Solution

```cpp
class Solution {
public:
    void moveZeroes(vector<int>& nums) {

        int k = 0;

        for (int i = 0; i < nums.size(); i++) {

            if (nums[i] != 0) {
                nums[k] = nums[i];
                k++;
            }
        }

        while (k < nums.size()) {
            nums[k] = 0;
            k++;
        }
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n)` |
| **Space Complexity** | `O(1)` |

---

## What I Learned

This problem helped me understand another application of the **Two Pointer** technique.

I used one pointer to traverse the array and another pointer to keep track of where the next non-zero element should be placed. After moving all non-zero elements to the front, I filled the remaining positions with zeros.

This approach modifies the array in-place without using any extra memory.

---

## Personal Notes

Initially, I thought about creating another array to store the non-zero elements and then appending zeros at the end. However, that approach uses additional space and doesn't satisfy the problem's requirement.

The two-pointer technique made the solution simple and efficient. This problem also reinforced the idea that many array modification problems can be solved in-place by carefully tracking the position where the next valid element should go.
