# 0303. Range Sum Query - Immutable

**Problem Name:** Range Sum Query - Immutable  
**Difficulty:** Easy

---

## Question

Given an integer array `nums`, implement a class that can efficiently answer multiple range sum queries.

For each query, return the sum of the elements between indices `left` and `right` (inclusive).

---

## My Solution

```cpp
class NumArray {
public:

    vector<int> prefix;

    NumArray(vector<int>& nums) {

        prefix.push_back(0);

        for (int i = 0; i < nums.size(); i++) {
            prefix.push_back(prefix.back() + nums[i]);
        }
    }

    int sumRange(int left, int right) {

        return prefix[right + 1] - prefix[left];
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Constructor** | `O(n)` |
| **sumRange()** | `O(1)` |
| **Space Complexity** | `O(n)` |

---

## What I Learned

This problem introduced me to the **Prefix Sum** technique.

Instead of calculating the sum for every query using a loop, I precomputed the cumulative sum of the array. Once the prefix sum array was built, any range sum could be calculated in constant time using a simple subtraction.

This significantly improves performance when there are many queries.

---

## Personal Notes

Initially, I thought about looping from `left` to `right` for every query. While that works, it takes `O(n)` time for each query and becomes slow when there are many queries.

Learning the Prefix Sum technique showed me how spending a little time during preprocessing can make future queries extremely fast. This is a useful pattern that I can apply to many array problems.
