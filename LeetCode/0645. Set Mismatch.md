# 0645. Set Mismatch

**Problem Name:** Set Mismatch  
**Difficulty:** Easy

---

## Question

You are given an integer array `nums` representing a set containing numbers from `1` to `n`.

Due to an error:
- One number appears **twice**.
- One number is **missing**.

Return the duplicated number and the missing number as an array.

---

## My Solution

```cpp
class Solution {
public:
    vector<int> findErrorNums(vector<int>& nums) {

        sort(nums.begin(), nums.end());

        int duplicate = -1;
        int missing = 1;

        for (int i = 1; i < nums.size(); i++) {

            if (nums[i] == nums[i - 1]) {
                duplicate = nums[i];
            }

            else if (nums[i] > nums[i - 1] + 1) {
                missing = nums[i - 1] + 1;
            }
        }

        if (nums[nums.size() - 1] != nums.size()) {
            missing = nums.size();
        }

        return {duplicate, missing};
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n log n)` |
| **Space Complexity** | `O(1)` *(Ignoring the space used by sorting.)* |

---

## What I Learned

This problem taught me how sorting can help identify duplicate and missing elements.

After sorting the array, duplicate numbers become adjacent, making them easy to detect. Similarly, if the difference between two consecutive numbers is greater than one, the missing number lies between them.

I also learned to handle the edge case where the missing number is the last number in the range.

---

## Personal Notes

Initially, I considered using a frequency array to count occurrences of each number. While that approach works, sorting provided a simpler solution without requiring additional memory.

The key insight was recognizing that sorting arranges the numbers in order, allowing both the duplicate and missing values to be identified in a single traversal.
