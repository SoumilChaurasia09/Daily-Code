# 0704. Binary Search

**Problem Name:** Binary Search  
**Difficulty:** Easy

---

## Question

Given a sorted integer array `nums` and an integer `target`, return the index of `target` if it exists in the array. Otherwise, return `-1`.

The solution must have a runtime complexity of **O(log n)**.

---

## My Solution

```cpp
class Solution {
public:
    int search(vector<int>& nums, int target) {

        int left = 0;
        int right = nums.size() - 1;

        while (left <= right) {

            int mid = left + (right - left) / 2;

            if (nums[mid] == target) {
                return mid;
            }

            else if (nums[mid] < target) {
                left = mid + 1;
            }

            else {
                right = mid - 1;
            }
        }

        return -1;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(log n)` |
| **Space Complexity** | `O(1)` |

---

## What I Learned

This problem helped me understand the Binary Search algorithm.

Instead of checking every element one by one, Binary Search repeatedly divides the search space into two halves. By comparing the middle element with the target, I can decide whether to search the left half or the right half of the array.

This approach is much faster than a linear search for sorted arrays.

---

## Personal Notes

Initially, I thought about using a simple loop to search for the target. Although that approach works, it takes `O(n)` time and doesn't satisfy the problem's requirement.

Learning Binary Search showed me how taking advantage of a sorted array can significantly improve performance. It also reinforced the importance of choosing the right algorithm based on the properties of the input.
