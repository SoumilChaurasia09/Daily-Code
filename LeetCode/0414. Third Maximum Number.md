# 0414. Third Maximum Number

**Problem Name:** Third Maximum Number  
**Difficulty:** Easy

---

## Question

Given an integer array `nums`, return the **third distinct maximum** number in the array. If the third distinct maximum does not exist, return the **maximum** number.

---

## My Solution

```cpp
class Solution {
public:
    int thirdMax(vector<int>& nums) {

        sort(nums.begin(), nums.end());

        vector<int> uniqueNums;

        for (int i = nums.size() - 1; i >= 0; i--) {

            if (uniqueNums.empty() || uniqueNums.back() != nums[i]) {
                uniqueNums.push_back(nums[i]);
            }

            if (uniqueNums.size() == 3) {
                return uniqueNums[2];
            }
        }

        return uniqueNums[0];
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n log n)` |
| **Space Complexity** | `O(n)` |

---

## What I Learned

This problem taught me the importance of handling **distinct** values instead of simply finding the third largest element.

After sorting the array, I traversed it from the end and stored only unique values. As soon as I found three distinct numbers, I returned the third one.

If fewer than three distinct numbers existed, I returned the largest number.

---

## Personal Notes

My first idea was to sort the array and directly return the third element from the end. However, I realized that duplicate values should only be counted once.

Keeping only distinct values after sorting made the solution simple and easy to understand. This problem also reminded me to carefully read the word **"distinct"** in the problem statement, as it completely changes the approach.
