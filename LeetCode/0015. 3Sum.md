# 0015. 3Sum

**Problem Name:** 3Sum  
**Difficulty:** Medium

---

## Question

Given an integer array `nums`, return all the unique triplets `[nums[i], nums[j], nums[k]]` such that:

- `i != j`
- `i != k`
- `j != k`
- `nums[i] + nums[j] + nums[k] == 0`

The solution must not contain duplicate triplets.

---

## My Solution

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {

        vector<vector<int>> ans;

        sort(nums.begin(), nums.end());

        int n = nums.size();

        for (int i = 0; i < n - 2; i++) {

            if (i > 0 && nums[i] == nums[i - 1])
                continue;

            int left = i + 1;
            int right = n - 1;

            while (left < right) {

                int sum = nums[i] + nums[left] + nums[right];

                if (sum == 0) {

                    ans.push_back({nums[i], nums[left], nums[right]});

                    while (left < right && nums[left] == nums[left + 1])
                        left++;

                    while (left < right && nums[right] == nums[right - 1])
                        right--;

                    left++;
                    right--;
                }

                else if (sum < 0) {
                    left++;
                }

                else {
                    right--;
                }
            }
        }

        return ans;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n²)` |
| **Space Complexity** | `O(1)` *(Ignoring the output array)* |

---

## What I Learned

This problem introduced me to combining **sorting** with the **two-pointer technique**.

After sorting the array, I fixed one element and used two pointers to search for the remaining two elements that make the sum equal to zero. I also learned how important it is to skip duplicate values to avoid repeated triplets in the final answer.

This approach is much more efficient than checking every possible triplet.

---

## Personal Notes

My first idea was to use three nested loops, but that would result in a time complexity of `O(n³)`, which is too slow for larger inputs.

The combination of sorting and two pointers reduced the complexity to `O(n²)`. Understanding how the `left` and `right` pointers move based on the current sum was the most valuable part of solving this problem.

This was also my first medium-level problem that required combining multiple concepts into one solution.
