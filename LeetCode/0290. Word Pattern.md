# 0290. Word Pattern

**Problem Name:** Word Pattern  
**Difficulty:** Easy

---

## Question

Given a pattern and a string `s`, determine whether the string follows the same pattern.

A valid pattern means:

- Each character in the pattern maps to exactly one word.
- Each word maps to exactly one character.
- No two characters map to the same word, and no two words map to the same character.

---

## My Solution

```cpp
class Solution {
public:
    bool wordPattern(string pattern, string s) {

        vector<string> words;
        string word = "";

        for (int i = 0; i < s.size(); i++) {

            if (s[i] == ' ') {
                words.push_back(word);
                word = "";
            }
            else {
                word += s[i];
            }
        }

        words.push_back(word);

        if (pattern.size() != words.size()) {
            return false;
        }

        unordered_map<char, string> mp1;
        unordered_map<string, char> mp2;

        for (int i = 0; i < pattern.size(); i++) {

            if (mp1.count(pattern[i]) == 0 && mp2.count(words[i]) == 0) {
                mp1[pattern[i]] = words[i];
                mp2[words[i]] = pattern[i];
            }

            else {

                if (mp1[pattern[i]] != words[i] || mp2[words[i]] != pattern[i]) {
                    return false;
                }
            }
        }

        return true;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n)` |
| **Space Complexity** | `O(n)` |

Where `n` is the length of the input string.

---

## What I Learned

This problem helped me understand how to maintain a **one-to-one mapping (bijection)** using two hash maps.

One hash map stores the mapping from pattern characters to words, while the other stores the mapping from words back to pattern characters. This ensures that no two characters map to the same word and no two words map to the same character.

---

## Personal Notes

Initially, I thought one hash map would be enough. However, I realized that using only one map cannot detect cases where two different characters map to the same word.

Using two hash maps made the relationship between characters and words clear and ensured that the mapping remained consistent in both directions.
