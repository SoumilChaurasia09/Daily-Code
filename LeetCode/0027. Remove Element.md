# 0027. Remove Element

**Problem Name:** Remove Element  
**Difficulty:** Easy

---

## Question

Given an integer array `nums` and an integer `val`, remove all occurrences of `val` **in-place** and return the number of elements that are not equal to `val`.

The order of the remaining elements does not matter. The first `k` elements of the array should contain all elements that are not equal to `val`.

---

## My Solution

```cpp
class Solution {
public:
    int removeElement(vector<int>& nums, int val) {

        int k = 0;

        for (int i = 0; i < nums.size(); i++) {

            if (nums[i] != val) {
                nums[k] = nums[i];
                k++;
            }

        }

        return k;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n)` |
| **Space Complexity** | `O(1)` |

---

## What I Learned

This problem helped me understand the **Two Pointer** technique more clearly.

Instead of creating a new array, I used one pointer to traverse the array and another pointer to keep track of where the next valid element should be placed. This allowed me to remove unwanted elements while modifying the original array in-place.

It also reinforced the idea that extra space isn't always necessary when the problem allows modifying the input array.

---

## Personal Notes

At first, I considered creating another array to store the required elements. However, after reading the problem carefully, I realized that the array itself could be reused.

The interesting part of this problem was that the order of the remaining elements wasn't important, but the simple two-pointer approach still preserved the original order.

This problem strengthened my understanding of in-place array manipulation and showed how a simple pointer can make array operations much more efficient.
