# 0035. Search Insert Position

**Problem Name:** Search Insert Position  
**Difficulty:** Easy

---

## Question

Given a sorted array of distinct integers `nums` and a target value, return the index if the target is found. If not, return the index where it would be inserted while maintaining the sorted order.

The solution must have a runtime complexity of **O(log n)**.

---

## My Solution

```cpp
class Solution {
public:
    int searchInsert(vector<int>& nums, int target) {

        int left = 0;
        int right = nums.size() - 1;

        while (left <= right) {

            int mid = left + (right - left) / 2;

            if (nums[mid] == target) {
                return mid;
            }

            else if (nums[mid] < target) {
                left = mid + 1;
            }

            else {
                right = mid - 1;
            }

        }

        return left;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(log n)` |
| **Space Complexity** | `O(1)` |

---

## What I Learned

This problem helped me understand the fundamentals of **Binary Search**.

Instead of checking every element one by one, Binary Search repeatedly divides the search space into two halves. This significantly reduces the number of comparisons and achieves a logarithmic time complexity.

I also learned that even when the target is not present, Binary Search naturally points to the correct insertion position by returning the `left` pointer after the loop ends.

---

## Personal Notes

At first, I thought of using a simple linear search, but that would take `O(n)` time and wouldn't satisfy the problem's requirement.

The most interesting part was realizing that after the search ends, `left` automatically represents the position where the target should be inserted.

This problem gave me a better understanding of how Binary Search works and why it is one of the most efficient algorithms for searching in sorted arrays.
