# 0455. Assign Cookies

**Problem Name:** Assign Cookies  
**Difficulty:** Easy

---

## Question

Each child has a greed factor, and each cookie has a size.

A child can receive at most one cookie, and a cookie can be given to only one child.

A child is satisfied if the cookie size is greater than or equal to the child's greed factor.

Return the maximum number of satisfied children.

---

## My Solution

```cpp
class Solution {
public:
    int findContentChildren(vector<int>& g, vector<int>& s) {

        sort(g.begin(), g.end());
        sort(s.begin(), s.end());

        int i = 0;
        int j = 0;

        while (i < g.size() && j < s.size()) {

            if (s[j] >= g[i]) {
                i++;
            }

            j++;
        }

        return i;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n log n + m log m)` |
| **Space Complexity** | `O(1)` *(Ignoring the space used by the sorting algorithm.)* |

Where:
- `n` = Number of children
- `m` = Number of cookies

---

## What I Learned

This problem introduced me to the **Greedy Algorithm**.

After sorting both the greed factors and cookie sizes, I always tried to satisfy the child with the smallest greed using the smallest possible cookie that could satisfy them. This greedy choice ensures that larger cookies remain available for children with larger greed factors.

---

## Personal Notes

Initially, I thought about trying every possible assignment between children and cookies, but that would be inefficient.

Sorting both arrays made the solution much simpler. By using two pointers, I could match children and cookies in a single pass after sorting. This problem helped me understand how greedy algorithms make locally optimal choices that lead to an optimal overall solution.
