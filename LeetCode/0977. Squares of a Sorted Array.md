# 0977. Squares of a Sorted Array

**Problem Name:** Squares of a Sorted Array  
**Difficulty:** Easy

---

## Question

Given a sorted integer array `nums`, return an array containing the square of each number, also sorted in non-decreasing order.

---

## My Solution

```cpp
class Solution {
public:
    vector<int> sortedSquares(vector<int>& nums) {

        int n = nums.size();
        vector<int> ans(n);

        int left = 0;
        int right = n - 1;
        int index = n - 1;

        while (left <= right) {

            if (abs(nums[left]) > abs(nums[right])) {
                ans[index] = nums[left] * nums[left];
                left++;
            }
            else {
                ans[index] = nums[right] * nums[right];
                right--;
            }

            index--;
        }

        return ans;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n)` |
| **Space Complexity** | `O(n)` |

---

## What I Learned

This problem taught me how the **Two Pointer** technique can be used with sorted arrays.

Although the original array is sorted, squaring negative numbers changes their order. Instead of sorting again, I compared the absolute values at both ends of the array. The larger square was placed at the end of the result array, and the corresponding pointer was moved inward.

This approach avoids sorting and achieves linear time complexity.

---

## Personal Notes

My first idea was to square every element and then sort the resulting array. While that solution works, it takes `O(n log n)` time.

The follow-up encouraged me to think differently. Using two pointers was a clever optimization that made the solution run in `O(n)` time. This problem helped me understand that the largest square always comes from one of the two ends of the sorted array.
