# 0575. Distribute Candies

**Problem Name:** Distribute Candies  
**Difficulty:** Easy

---

## Question

Alice has `n` candies, where each candy belongs to a certain type. She is allowed to eat only `n / 2` candies.

Return the maximum number of **different candy types** she can eat.

---

## My Solution

```cpp
class Solution {
public:
    int distributeCandies(vector<int>& candyType) {

        unordered_set<int> types;

        for (int i = 0; i < candyType.size(); i++) {
            types.insert(candyType[i]);
        }

        int totalTypes = types.size();
        int canEat = candyType.size() / 2;

        return min(totalTypes, canEat);
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n)` |
| **Space Complexity** | `O(n)` |

---

## What I Learned

This problem helped me understand how an **unordered_set** can be used to count distinct elements efficiently.

After finding the total number of unique candy types, I compared it with the maximum number of candies Alice is allowed to eat (`n / 2`). The answer is simply the smaller of these two values.

---

## Personal Notes

Initially, I thought about counting the frequency of every candy type. After reading the problem carefully, I realized that only the number of **unique** candy types matters, not how many times each type appears.

Using an `unordered_set` made the solution very simple. This problem reinforced the idea that sets are useful whenever we need to work with distinct values.
