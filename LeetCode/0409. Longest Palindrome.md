# 0409. Longest Palindrome

**Problem Name:** Longest Palindrome  
**Difficulty:** Easy

---

## Question

Given a string `s` consisting of lowercase and uppercase English letters, return the length of the longest palindrome that can be built using those letters.

Note that uppercase and lowercase letters are treated as different characters.

---

## My Solution

```cpp
class Solution {
public:
    int longestPalindrome(string s) {

        unordered_map<char, int> mp;

        for (char ch : s) {
            mp[ch]++;
        }

        int length = 0;
        bool odd = false;

        for (auto it : mp) {

            if (it.second % 2 == 0) {
                length += it.second;
            }
            else {
                length += it.second - 1;
                odd = true;
            }
        }

        if (odd) {
            length++;
        }

        return length;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n)` |
| **Space Complexity** | `O(1)` *(At most 52 different English letters.)* |

---

## What I Learned

This problem helped me understand how **character frequency counting** can be used to build a palindrome.

A palindrome can contain all characters that appear an even number of times. For characters with odd frequencies, I can use all but one of their occurrences. If there is at least one character with an odd frequency, one of those leftover characters can be placed in the center of the palindrome.

---

## Personal Notes

Initially, I thought I needed to actually construct the palindrome. After reading the problem carefully, I realized that only the maximum possible length was required.

Using a hash map to count character frequencies made the solution straightforward. This problem also reinforced the property that a palindrome can have **at most one character with an odd frequency**, which becomes the center of the palindrome.
