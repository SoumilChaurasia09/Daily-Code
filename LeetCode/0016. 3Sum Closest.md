# 0016. 3Sum Closest

**Problem Name:** 3Sum Closest  
**Difficulty:** Medium

---

## Question

Given an integer array `nums` and an integer `target`, find three integers whose sum is closest to the target.

Return the sum of those three integers.

You may assume that exactly one solution exists.

---

## My Solution

```cpp
class Solution {
public:
    int threeSumClosest(vector<int>& nums, int target) {

        sort(nums.begin(), nums.end());

        int closest = nums[0] + nums[1] + nums[2];

        for (int i = 0; i < nums.size() - 2; i++) {

            int left = i + 1;
            int right = nums.size() - 1;

            while (left < right) {

                int sum = nums[i] + nums[left] + nums[right];

                if (abs(target - sum) < abs(target - closest)) {
                    closest = sum;
                }

                if (sum < target) {
                    left++;
                }
                else if (sum > target) {
                    right--;
                }
                else {
                    return sum;
                }
            }
        }

        return closest;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n²)` |
| **Space Complexity** | `O(1)` *(Ignoring the space used by sorting.)* |

---

## What I Learned

This problem helped me strengthen my understanding of the **Two Pointer** technique.

After sorting the array, I fixed one element and used two pointers to search for the other two elements. Instead of looking for an exact sum of zero like in the 3Sum problem, I kept track of the sum that was closest to the target.

Whenever I found a sum closer to the target than the previous best, I updated the answer.

---

## Personal Notes

At first, I thought about checking every possible triplet using three nested loops. While that would work, it would take `O(n³)` time and be too slow.

Using sorting together with two pointers reduced the complexity to `O(n²)`. This problem also helped me understand how a small change in the problem statement can lead to a different use of the same algorithm.
