# 0108. Convert Sorted Array to Binary Search Tree

**Problem Name:** Convert Sorted Array to Binary Search Tree  
**Difficulty:** Easy

---

## Question

Given a sorted integer array `nums`, convert it into a **height-balanced Binary Search Tree (BST)**.

A height-balanced BST is a binary tree in which the depth of the two subtrees of every node never differs by more than one.

---

## My Solution

```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right)
 *         : val(x), left(left), right(right) {}
 * };
 */

class Solution {
public:

    TreeNode* build(vector<int>& nums, int left, int right) {

        if (left > right) {
            return nullptr;
        }

        int mid = (left + right) / 2;

        TreeNode* root = new TreeNode(nums[mid]);

        root->left = build(nums, left, mid - 1);
        root->right = build(nums, mid + 1, right);

        return root;
    }

    TreeNode* sortedArrayToBST(vector<int>& nums) {

        return build(nums, 0, nums.size() - 1);

    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n)` |
| **Space Complexity** | `O(log n)` *(Recursive call stack for a balanced tree)* |

## What I Learned

This problem introduced me to the concept of building a **height-balanced Binary Search Tree** using **recursion**.

The key idea is to always choose the middle element of the sorted array as the root node. This naturally keeps the left and right subtrees balanced.

I also learned how recursion can simplify tree construction by solving the same problem for the left and right halves of the array.

## Personal Notes

Initially, I wasn't sure how to build a balanced BST from a sorted array. After understanding the recursive approach, I realized that choosing the middle element at every step automatically keeps the tree balanced.

This was also my first problem involving recursive tree construction, and it helped me understand how divide-and-conquer can be applied to binary trees.

Although the implementation is short, the recursive thinking behind it was the biggest takeaway from this problem.
