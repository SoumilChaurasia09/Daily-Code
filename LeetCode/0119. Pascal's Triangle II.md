# 0119. Pascal's Triangle II

**Problem Name:** Pascal's Triangle II  
**Difficulty:** Easy

---

## Question

Given an integer `rowIndex`, return the `rowIndex`th row (0-indexed) of Pascal's Triangle.

The follow-up asks for a solution that uses only `O(rowIndex)` extra space.

---

## My Solution

```cpp
class Solution {
public:
    vector<int> getRow(int rowIndex) {

        vector<int> row(rowIndex + 1, 1);

        for (int i = 2; i <= rowIndex; i++) {

            for (int j = i - 1; j > 0; j--) {
                row[j] = row[j] + row[j - 1];
            }

        }

        return row;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n²)` |
| **Space Complexity** | `O(n)` |

Where `n = rowIndex`.

---

## What I Learned

This problem helped me understand how Pascal's Triangle can be generated using only a single array.

Instead of creating the entire triangle, I updated one row in-place. By updating the array from **right to left**, the previous values remained available for calculating the next element.

This approach satisfies the follow-up requirement of using only `O(rowIndex)` extra space.

---

## Personal Notes

Initially, I considered generating the entire Pascal's Triangle and then returning the required row. Although that works, it uses unnecessary extra memory.

Updating a single row from right to left was a clever optimization. This problem showed me that sometimes the same data structure can be reused to save space without changing the overall logic.
