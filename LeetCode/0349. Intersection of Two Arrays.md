# 0349. Intersection of Two Arrays

**Problem Name:** Intersection of Two Arrays  
**Difficulty:** Easy

---

## Question

Given two integer arrays `nums1` and `nums2`, return an array containing their intersection.

Each element in the result must be **unique**, and the order of the output does not matter.

---

## My Solution

```cpp
class Solution {
public:
    vector<int> intersection(vector<int>& nums1, vector<int>& nums2) {

        unordered_set<int> s1(nums1.begin(), nums1.end());
        unordered_set<int> ans;

        for (int i = 0; i < nums2.size(); i++) {

            if (s1.count(nums2[i])) {
                ans.insert(nums2[i]);
            }
        }

        return vector<int>(ans.begin(), ans.end());
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(n + m)` |
| **Space Complexity** | `O(n)` |

Where:
- `n` = Size of `nums1`
- `m` = Size of `nums2`

---

## What I Learned

This problem helped me understand how **unordered_set** can be used to efficiently find common elements between two arrays.

I stored all elements of the first array in a hash set for fast lookup. Then, I traversed the second array and checked whether each element existed in the set. Another set was used to ensure that the final answer contained only unique values.

This approach is much faster than comparing every pair of elements.

---

## Personal Notes

My first idea was to use two nested loops to compare every element of both arrays. Although that works, it has a time complexity of `O(n × m)`, which is inefficient for larger inputs.

Using hash sets made the solution much cleaner and reduced the complexity to linear time. This problem also reinforced how useful sets are when uniqueness is required.
