# 0405. Convert a Number to Hexadecimal

**Problem Name:** Convert a Number to Hexadecimal  
**Difficulty:** Easy

---

## Question

Given a 32-bit integer `num`, return its hexadecimal representation as a string.

- Use lowercase letters (`a` to `f`).
- Do not include leading zeros.
- For negative numbers, use two's complement representation.
- Do not use any built-in method that directly converts numbers to hexadecimal.

---

## My Solution

```cpp
class Solution {
public:
    string toHex(int num) {

        if (num == 0)
            return "0";

        string hex = "0123456789abcdef";
        string ans = "";

        unsigned int n = num;

        while (n > 0) {

            ans = hex[n % 16] + ans;

            n /= 16;
        }

        return ans;
    }
};
```

---

## Complexity

| Metric | Value |
|--------|-------|
| **Time Complexity** | `O(1)` *(At most 8 hexadecimal digits for a 32-bit integer)* |
| **Space Complexity** | `O(1)` |

## What I Learned

This problem helped me understand how hexadecimal numbers are generated manually.

Instead of using a built-in conversion function, I repeatedly divided the number by `16` and used the remainder to find the corresponding hexadecimal character.

I also learned that converting the integer to an `unsigned int` automatically handles negative numbers using two's complement representation.

---

## Personal Notes

Initially, I thought negative numbers would require special handling. After learning about `unsigned int`, I realized that C++ already stores negative integers using two's complement, making the conversion much simpler.

This problem also gave me a better understanding of number systems and how decimal values are represented in hexadecimal.
